Core Frame
What Big Idea 2 Is Really About

Big Idea 2 builds the bridge between raw bits and meaningful information. Everything a computer handles — text, images, audio, video — begins as patterns of binary (0s and 1s). Understanding how data is represented, compressed, and analyzed is central to AP CSP.

At its core, Big Idea 2 asks: how do computers store information, how do we reduce it efficiently, and how do we extract knowledge from massive datasets?

Common Mistakes

Avoid These

  • Thinking all compression reduces quality — only lossy compression does; lossless preserves everything.
  • Confusing number of bits with the maximum value — 8 bits maxes at 255, not 8.
  • Treating correlation as proof of causation without checking for confounding factors.
  • Assuming "more data = better conclusions" — biased data stays biased regardless of volume.
  • Forgetting that removing obvious identifiers does not fully anonymize data; metadata can still expose individuals.

Major Buckets
01 · Binary Numbers

Why Everything Is 0s and 1s

A bit is a single binary digit (0 or 1). A byte is 8 bits. Computer hardware reliably distinguishes only two physical states (high/low voltage), so binary is the universal language of computing — connecting hardware, software, and networks.

With n bits, there are 2n possible patterns. More bits means more patterns — which can represent bigger numbers, more characters, more colors, and so on.

1 bit
21 = 2 patterns: 0 or 1
8 bits (1 byte)
28 = 256 patterns: 0–255
Overflow
When a value exceeds the maximum for a fixed number of bits, the result "wraps around" and becomes incorrect. With 8 bits, max is 255; trying to store 256 causes overflow.
Key Misconception
More bits means more patterns — not just bigger numbers. It can mean more colors, more characters, more precision in audio.
02 · Binary ↔ Decimal Conversion

Reading and Writing Binary

Binary is a base-2 positional system. Each position represents a power of 2, counted right to left starting at 20 = 1.

Place Values
…64 | 32 | 16 | 8 | 4 | 2 | 1 (right to left, each doubles).
Binary → Decimal
Add up place values where there's a 1. Example: 10110 → 16+4+2 = 22.
Decimal → Binary
Subtract the largest power of 2 that fits, record a 1, repeat. Example: 37 → 32+4+1 = 100101.
Common Error
Don't treat binary digits as base-10 digits. You must use powers-of-2 place values.

Worked Example: 10110₂

Positions: 16 8 4 2 1 Bits: 1 0 1 1 0 Value: 16 + 0 + 4 + 2 + 0 = 22

Worked Example: 37₁₀ → Binary

37 - 32 = 5 → place 1 at 32 5 - 4 = 1 → place 1 at 4 1 - 1 = 0 → place 1 at 1 Zeros at 16, 8, 2 → result: 1 0 0 1 0 1
03 · Encoding Data Types

How Text, Images, and Sound Become Bits

Binary doesn't only represent numbers. Any data can be encoded as bits as long as both sender and receiver agree on the encoding scheme.

Text
Each character is assigned a number (like ASCII or Unicode), then stored in binary. If sender and receiver use different mappings, characters display incorrectly.
Images
Made of pixels. Each pixel's color is stored in bits. Color depth (bits per pixel) determines how many colors are possible: b bits → 2b colors. Higher depth + higher resolution = larger file.
Audio
Captured by sampling: measuring air pressure at regular intervals. Sample rate (samples/sec) and bit depth (bits/sample) control quality and file size.
AP Move
Higher quality always means larger files. This creates the tradeoff that motivates compression.
04 · Data Compression

Lossless vs. Lossy — The Critical Distinction

Data compression encodes information using fewer bits. It matters for storage (how much fits on a device), speed (how fast files transfer), and cost. But compression trades size against compute time — and sometimes quality.

Lossless
Reduces size while preserving all data. Decompressing gives back the exact original. Best for: text, code, legal documents, any data where every bit matters. Idea: find and eliminate redundancy.
Lossy
Permanently removes information (usually details humans barely notice). Smaller files, but not reversible. Best for: photos, audio, video — where "good enough" quality is acceptable.
RLE (Run-Length Encoding)
A lossless technique. Stores "value + count" instead of repeating values. Works well for data with long repeated runs; can backfire if data changes frequently.
Dictionary Compression
Replaces repeated sequences with short references. The dictionary is stored alongside compressed data so the original can be perfectly reconstructed.

Choosing Between Lossless and Lossy

Ask: does every bit matter? Medical images, blueprints, legal text → lossless. Streaming video, social media photos → lossy is acceptable to save bandwidth. Compression never magically removes meaning; it either changes the representation (lossless) or discards some information (lossy).

05 · Extracting Information from Data

Turning Raw Data Into Insight

Modern computing generates massive datasets. Extracting information means using computational tools to find patterns, test hypotheses, and support decisions. But data analysis has risks: bias, misleading summaries, and confusing correlation with causation.

Summaries & Aggregation
Counts, totals, averages, min/max. An average can hide important variation — two groups can have the same mean but very different distributions.
Trends, Clusters, Outliers
Computers scan large datasets for patterns humans would miss. Outliers can be errors or the most important data points — always investigate before deleting.
Visualization
Charts and graphs reveal patterns quickly, but they can mislead if axes are manipulated or context is missing. Same data → different graphs → different impressions.
Correlation ≠ Causation
Two variables can move together because of a third factor, coincidence, or reversed causation. Example: ice cream sales and drowning rise together because of hot weather — not because ice cream causes drowning.

Sampling and Bias

When you can't collect all data, you use a sample. Sampling bias occurs when certain groups are systematically excluded. More data does not fix bias — a huge biased dataset is still biased.

06 · Using Programs with Data

The Analysis Pipeline

Programs enable data analysis at scale: a person can't inspect millions of rows, but a program can. Programs are fast and consistent, but they follow your instructions literally — mistakes in code or data produce wrong results without any error.

Filter
Keep only rows matching a condition. Critical distinction: filtering keeps data that matches; it does not change values.
Count vs. Sum
Counting increments by 1 per match. Summing adds the value itself. A common exam trap: confusing a loop that counts items with one that sums them.
Average
Requires both total sum and count. Missing/invalid entries can silently corrupt the result unless you clean data first.
Exact String Matching
Programs match categories exactly. "NY" ≠ "New York" ≠ "ny". This is why cleaning is a prerequisite — inconsistent strings split counts that should be combined.

Pseudocode: Filtering and Counting

temps ← [72, 68, 75, 70, 80, 65] countHot ← 0 FOR EACH t IN temps IF t > 72 countHot ← countHot + 1 DISPLAY(countHot) // outputs 2
07 · Metadata

Data About Data

Metadata is information that describes other data — it provides context that makes a dataset interpretable and usable. Without metadata, a column of numbers is meaningless.

Descriptive Metadata
Identifies and explains the data: column names, descriptions, keywords, dataset title.
Structural Metadata
How data is organized: data types, table schema, file format, relationships between tables.
Administrative Metadata
Rules and provenance: who collected it, when updated, licensing, access permissions, privacy notes.
Privacy Risk
Metadata can reveal sensitive information even when obvious identifiers are removed. A photo's GPS coordinates, timestamps, or camera model can re-identify individuals or reveal locations.

AP Move on Metadata

On exam questions: identify what metadata is present or missing, explain how its absence makes interpretation unreliable, and reason about whether metadata creates privacy risk. Coded categories (1, 2, 3) require metadata to interpret — computing a numerical average of category codes produces meaningless results.

08 · Cleaning Data

Making Data Fit for Analysis

Cleaning data means detecting and resolving problems so a dataset can be reliably analyzed. Real-world data is messy — it comes from people typing, sensors glitching, and systems merging. Cleaning is defined by metadata: what "clean" means depends on what the data should represent.

Missing Values
Options: exclude from calculations, impute (fill with estimate), or remove rows. Each approach has tradeoffs — dropping rows can bias results if missing values cluster in one group.
Inconsistent Categories
"CA", "ca", "California" must be standardized before grouping. Programs compare exact strings — inconsistency silently splits counts.
Duplicate Records
Sometimes errors (system glitch), sometimes real repeated events (multiple measurements). Metadata about collection method helps you decide whether to remove them.
Mixed Types & Impossible Values
Numeric column containing text, negative ages, temperatures of 999. Require validation rules and investigation — not automatic deletion.

Common Cleaning Pitfalls

Treating missing values as zeros (drastically skews averages), deleting outliers without investigating them, and assuming the program ran so the data must be fine. Many data errors produce plausible-looking but wrong results — no crash, no warning.

Exam Prep

AP Exam Strategies

High-yield patterns and common traps for Big Idea 2

Binary & Representation

High-Yield Patterns

  • Given n bits, how many distinct values? → 2n
  • Given x values needed, minimum bits? → smallest n where 2n ≥ x
  • Convert binary to decimal using place values (don't guess).
  • Max value with n bits = 2n − 1 (not 2n).
  • Overflow: value exceeds storage capacity → result wraps or corrupts.
Compression

Lossless vs. Lossy

  • Lossless: exact recovery, good for text/code/legal. Exploits redundancy.
  • Lossy: smaller files, not reversible, good for photos/video/audio.
  • RLE example: 111111000011 → "6 ones, 4 zeros, 2 ones".
  • Compression may increase compute time even while reducing file size.
  • Compression cannot always make files smaller — if data has no redundancy, lossless may fail to compress.
Data Analysis

Reasoning About Data

  • Correlation ≠ Causation — always ask: is there a third factor? Is the direction reversed?
  • Sampling bias: missing groups → biased conclusions regardless of sample size.
  • Outliers: investigate before removing. May be the most important data points.
  • Visualization can mislead: check axis scales and missing context.
  • Averages hide variation — two datasets can have identical means with very different distributions.
Metadata & Privacy

Privacy Implications

  • Metadata (timestamps, GPS, camera model) can re-identify individuals even after names are removed.
  • Combining datasets can reveal information neither dataset disclosed alone.
  • Coded categories (1/2/3) should not be treated as quantities — metadata tells you what they mean.
  • A "blank" can mean missing, not applicable, or redacted — metadata defines the difference.
Programs with Data

Trace and Predict Output

  • Know the difference between counting (+1 each time) and summing (+value each time).
  • Filters keep matching rows — they don't change values.
  • Programs match strings exactly — "NY" ≠ "New York".
  • Invalid/missing data in a list can silently corrupt averages.
  • A program running without an error does not mean the output is correct.
Cleaning Data

Cleaning Decision Tradeoffs

  • Dropping rows with missing data is only safe if missing data is random and rare.
  • If missing data clusters in one group, dropping those rows biases the dataset.
  • Imputing values (filling with estimates) hides uncertainty and can distort distributions.
  • Standardizing categories is required before grouping/counting — "ca" and "CA" are different strings.
  • Document all cleaning changes as new metadata so others can interpret results correctly.
